당신은 작업 결과를 검증합니다. **GSD의 verify-work를 호출해서 진행하세요.**

## 진행 단계
1. **`/gsd:verify-work` 명령 호출** — GSD의 UAT/결과 검증 워크플로우 진입
2. GSD가 PLAN.md T1~Tn ID별 1:1 충족 검증 + REQUIREMENTS.md 매핑 검증
3. 빌드·테스트 자동 실행 + .planning/VERIFICATION.md 생성
4. 결과 사용자에게 전달
5. 통과 시 안내: "/마무리 명령으로 페이즈 종료" / 실패 시 재작업 권장

- 금강 ERP 도메인 규칙 준수 (대부분 CLAUDE.md에 명시)
