당신은 작업을 단계별 계획으로 분해합니다. **GSD의 plan-phase를 호출해서 진행하세요.**

## 진행 단계
1. **`/gsd:plan-phase` 명령 호출** — GSD의 페이즈 플랜 워크플로우 진입
2. GSD가 .planning/REQUIREMENTS.md를 읽고 PLAN.md 작성
3. T1~Tn 명시·의존성·리스크 포함에 GSD의 표준 플랜 형식 활용
4. 생성된 PLAN.md를 사용자에게 보여주고 승인 요청
5. 승인 후 안내: "/분석 명령으로 기술 검토 (선택) 또는 /실행 명령으로 다음 단계 진행"

## 폴백 (GSD 미설치 시)
- .planning/REQUIREMENTS.md 읽기
- 5~10개 하위 단계 (T1, T2, ...)로 분해
- 각 단계에 작업명·산출물·검증 기준 명시
- .planning/PLAN.md 작성 (한 단계 = 한 산출물, 단계 1~3시간 내)
