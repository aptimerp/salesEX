## 개요

SalesEX는 **입찰·견적·수주·계약·출고·정산·실적**으로 이어지는 건설/제조형 영업운영 플랫폼이다.

### 설계 철학
- **Flow First**: 업무 흐름 중심 — 각 단계가 자동으로 다음 단계에 연결
- **Input Once**: 한 번 입력한 데이터는 재사용 (엑셀 취합 제거)
- **Connected Workflow**: 단계 전환 시 자동 데이터 이관
- **Real-time Aggregation**: 실시간 자동 취합
- **Field-driven**: 현업 실무 흐름 기준 설계
- **Salesforce CRM 스타일 흐름 기반 구조** 참고

### 시스템 3계층 구조
```
[K-DUO LINK 모바일]  → 현장 영업활동 입력 (PWA)
       ↓
[SalesEX 웹 CRM]     → 영업 흐름 관리 / 취합 / 분석
       ↓
[ERP]                → 계약 / 출고 / 정산 / 회계
```

## 단계별 개발 계획

### 1단계 — CRM 기초 구축
> 목표: 엑셀 제거 시작 / 영업 기본 데이터 DB화

| 모듈 | 핵심 기능 |
|---|---|
| 1-1. 로그인/인증/조직관리 | 이메일+비밀번호 로그인, Supabase Auth, 권한 그룹, 사용자 CRUD, 팀/부서 관리 |
| 1-2. 현장관리 | 현장 목록(필터), 신규등록, 상세/수정, 상태변경, 활동이력 연결 |
| 1-3. 입찰관리 | 입찰 목록(필터), 등록(현장 연결), 경쟁사 정보, 현설 일정 알림 |
| 1-4. 견적관리 | 견적 목록, 등록(입찰 연결), Revision 관리, 단가 비교, 마진율 자동계산, 승인 흐름 |

### 2단계 — 수주/계약/보고 자동화
> 목표: 영업 전체 흐름 연결 / ERP 연계 / 주간보고 자동화

| 모듈 | 핵심 기능 |
|---|---|
| 2-1. 활동관리 | 활동 목록(필터), 등록(웹/모바일), 첨부파일, 차기 액션 알림 |
| 2-2. 수주관리 | 수주 등록(견적 연결), 낙찰/탈락/보류, 수주율, 팀별/담당자별 실적 |
| 2-3. 계약관리 | 계약 등록(수주 연결), ERP API 자동 전송, 중복 검증, 연계 로그 |
| 2-4. 출고/납기관리 | 납기 캘린더/리스트, 출고예정, 상태 업데이트, 납기 지연 알림 |
| 2-5. 수금관리 | 수금 현황, 미수금 자동계산, 수금 예정일 알림 |
| 2-6. 주간업무보고 자동화 | 주간/입찰/견적/수주/월간실적/사업계획 보고서 자동 생성 |

### 3단계 — 분석/대시보드/모바일
> 목표: 경영 의사결정 지원 / 실시간 KPI 관리

| 모듈 | 핵심 기능 |
|---|---|
| 3-1. 사업계획 관리 | 팀별 연간 계획, 품목/월별 목표, 달성률 자동계산, 전년 비교 |
| 3-2. 실적분석 대시보드 | 영업KPI(수주금액/수주율/출고), 조직KPI(팀/담당/지역/품목), 경영KPI |
| 3-3. 모바일 앱 (PWA) | 현장 미팅 로그, 현장 선등록, 명함 등록, 사진 첨부, 음성 메모 |

### 확장 단계 — 재임대·보수 (별도 마일스톤)
> K-DUO LINK 확장 + 자산관리 DB 신규 구축

- 회수 관리 (K-DUO LINK 회수팀 뷰)
- AS 관리 (K-DUO LINK AS팀 뷰)
- 보수 워크플로우 (쇼트→검수→등급→재출고)
- 자산 DB(DB-4) 구축 (착탈식 스테인리스 태그 → 홀 패턴 → NFC)
- 재임대·보수 KPI 대시보드

  │  PK: company_id

  ├─── persons (담당자)
  │      FK: company_id → companies
  │      ERP연동: person_code
  │
  ├─── sites (현장)

┌─ 실적 요약 탭 (자동 집계) ──────────────────┐
│  누적계약금액 / 계약건수 / 계약동수           │

## ERP 연동 설계

### ERP → CRM 동기화 (읽기)
| 대상 | 키 | 방식 |
|---|---|---|
| 거래처 (companies) | company_code | 배치 동기화 (권장) |
| 담당자 (persons) | person_code | 배치 동기화 |
| 현장 (sites) | site_code | 배치 동기화 |
| 출고/정산 데이터 | - | 계약 후 ERP → CRM 전송 |

### CRM → ERP 전송 (쓰기)
| 대상 | 트리거 | 전송 데이터 |
|---|---|---|
| 계약 | 수주확정 → 계약등록 완료 | 계약번호/계약금액/납기일/거래처코드/현장코드 |

### 연동 원칙
- 호출 경로: 클라이언트 → Vercel API Routes → 금강 API (서버 사이드 전용)
- API URL: 환경변수 `KUMKANG_API_URL`로만 사용 (하드코딩 금지)
- API 키: `KUMKANG_API_KEY` 서버 측에서만 사용 (클라이언트 노출 금지)
- 마스터 데이터: 배치 동기화 권장 (야간/주기적)
- 계약 전송: 실시간 API 권장
- 수동 동기화: 보조 수단 (담당자 버튼 클릭)
- 중복 방지: ERP 코드(company_code, site_code, person_code) 기준

## 현행 업무 → CRM 전환 흐름

### 영업 담당자 (폼팀/건축팀)
| 현행 (엑셀) | CRM 전환 후 |
|---|---|
| 주간보고서 현장명 기재 | 현장관리에서 현장 검색/선택 (ERP 동기화) |
| 주간보고서 입찰정보 기재 | 입찰 등록 화면에서 직접 입력 |
| 견적현황 엑셀 수기 계산 | 입찰 화면에서 단가·물량 → 자동계산 |
| 입찰 결과 수기 기재 | 결과 드롭다운 + 실패사유 분류 → 경쟁사 분석 자동화 |
| 주간보고서 특이사항 | 활동 이력 별도 등록 (차기액션, 경쟁사, 수주가능성) |

### 영업지원팀
| 현행 (엑셀/ERP) | CRM 전환 후 |
|---|---|
| 팀별 보고서 수동 취합 | CRM DB 기반 통합보고 자동 생성 |
| 수주잔고 수기 추가 | 수주확정 시 자동 생성 |
| ERP 계약 수동 입력 | CRM 계약 등록 → ERP 자동 전송 |
| 수금현황 엑셀 관리 | 수금관리 화면 (미수금 자동계산) |
